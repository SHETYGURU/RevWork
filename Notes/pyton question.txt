What is an Interpreter?

An Interpreter is a program that reads and executes code line by line.

How it works:
Takes one line of code
Translates it to machine language
Executes it immediately
Moves to next line

Advantages:
Easy to debug (error shown immediately)
No need to compile entire program
Faster development

Disadvantages:
Slower execution
Errors stop execution instantly

What is a Compiler?

A Compiler is a program that translates the entire code at once into machine code before execution.

How it works:
Takes whole program
Converts into machine code
Generates executable file
Then runs it
Example Languages:
C
C++
Go


Advantages:
Faster execution
Optimized performance
Errors shown after full compilation
Disadvantages:
Debugging is harder
Must compile again after changes

Purpose:
Handles the error if it occurs

What is a Python Function?

A function is a reusable block of code that performs a specific task.

Instead of writing the same code again and again, you define it once and reuse it.

Basic Syntax
def function_name(parameters):
    # code block
    return value
Example
def add(a, b):
    return a + b

result = add(10, 20)
print(result)   # 30
Types of Functions
1. Built-in Functions

Already available in Python

print("Hello")
len([1, 2, 3])
type(10)
2. User-defined Functions

Created by the user

def greet(name):
    print("Hello", name)
3. Lambda (Anonymous Function)

Small one-line function

add = lambda a, b: a + b
print(add(5, 3))
Types Based on Arguments & Return
Type	Example
No arguments, no return	def fun():
With arguments, no return	def fun(a):
No arguments, with return	def fun(): return 10
With arguments & return	def fun(a,b): return a+b
Function with Default Parameters

    print("Hello", name)

Keyword Arguments

Variable Length Arguments

**kwargs (key-value)

Return Statement

Sends result back to caller

Definition:

super() is used to call parent class methods/constructor.

Example:
class Parent:
    def __init__(self):
        print("Parent constructor")

    def __init__(self):
        super().__init__()
        print("Child constructor")
Use:

Access parent functionality
5. Inheritance
Definition:

Inheritance allows one class to reuse properties of another class.

Example:
class Animal:
    def sound(self):
        print("Some sound")

class Dog(Animal):
    pass

d = Dog()
d.sound()
Use:
Code reusability
Hierarchical structure
6. Polymorphism
Definition:

Polymorphism means same method name behaves differently.

Example:
class Dog:
    def sound(self):
        print("Bark")

class Cat:
    def sound(self):
        print("Meow")

for animal in [Dog(), Cat()]:
    animal.sound()
Use:
Flexible code
Same interface, different behavior
7. Method Overloading (Python Style)
Definition:

Same method name with different parameters (Python handles using default args).

Example:
def add(a, b=0, c=0):
    return a + b + c

print(add(2))
print(add(2,3))
print(add(2,3,4))
Use:
Handle multiple input scenarios
8. Method Overriding
Definition:

Child class redefines parent method.

Example:
class Parent:
    def show(self):
        print("Parent")

class Child(Parent):
    def show(self):
        print("Child")

c = Child()
c.show()
Use:
Customize behavior in child class
9. Abstraction
Definition:

Hiding implementation details and showing only essential features.

Example:
from abc import ABC, abstractmethod

class Vehicle(ABC):
    @abstractmethod
    def start(self):
        pass

class Car(Vehicle):
    def start(self):
        print("Car starts with key")
Use:
Enforce rules (must implement methods)
Cleaner design

What is Vectorization?

Vectorization means performing operations on entire arrays at once
without using loops (for/while)

Simple Idea

Instead of processing one element at a time,
you process all elements together

Without Vectorization (Slow )
arr = [1, 2, 3, 4]

result = []
for x in arr:
    result.append(x * 2)

print(result)

Uses loop → slower

With Vectorization (Fast)
import numpy as np

arr = np.array([1, 2, 3, 4])
result = arr * 2

print(result)

No loop → much faster

Visual Understanding
Why Vectorization is Important
1. High Performance
Uses optimized C-level implementation
Much faster than Python loops
2. Cleaner Code
Less code, more readable
3. Efficient Memory Usage
Avoids intermediate objects
More Examples
Addition
a = np.array([1,2,3])
b = np.array([4,5,6])

print(a + b)   # [5 7 9]
Square
arr = np.array([1,2,3])
print(arr ** 2)   # [1 4 9]
Conditional (Vectorized)
arr = np.array([10, 20, 30, 40])
print(arr[arr > 20])   # [30 40]
Vectorization vs Loop
Feature	Loop 	Vectorization
Speed	Slow	Fast
Code	More lines	Less lines
Performance	Low	High
Readability	Medium	High
Real-Time Use Cases
Data transformation (ETL pipelines)
Machine Learning preprocessing
Financial calculations
Image processing (pixel operations)

What is a Named Tuple?

A named tuple is a special type of tuple where each element has a name (field).

It is like a tuple + object (with attributes)

Simple Idea

Normal tuple:

person = ("John", 25)

Access using index → confusing

person[0]

Named tuple:

person.name

Easy to read and understand

How to Create Named Tuple
from collections import namedtuple

Person = namedtuple("Person", ["name", "age"])

p1 = Person("John", 25)

print(p1.name)   # John
print(p1.age)    # 25
Key Features
1. Named Fields

Access using names instead of index

2. Immutable

Like tuple → cannot modify values

# p1.name = "Sam" Error
3. Lightweight

Faster than class

Access Methods
print(p1[0])        # Index access
print(p1.name)      # Name access
Useful Methods
p1._asdict()     # Convert to dictionary
p1._replace(age=30)  # Create new object with updated value
Example (Real-Time)
Employee = namedtuple("Employee", ["id", "name", "salary"])

emp1 = Employee(101, "John", 50000)

print(emp1.name)
Why Use Named Tuple?
Readability

emp.name is better than emp[1]

Immutability

Data is safe (no accidental changes)

Performance

Uses less memory than class

Named Tuple vs Tuple vs Class

Named fields		✅	✅
Mutable			✅
Memory	Low	Low	High
Readability	Low	High	High

Here’s a clear + interview-ready explanation of Counter in Python 

What is Counter?

A Counter is a class from the built-in collections module used to count the frequency of elements.

It works like a dictionary (key → count)

Import Counter
from collections import Counter
Basic Example
from collections import Counter

data = [1, 2, 2, 3, 3, 3]

c = Counter(data)
print(c)

Output:

Counter({3: 3, 2: 2, 1: 1})
Simple Idea

Counter = Automatic frequency counter

Instead of writing:

freq = {}
for i in data:
    freq[i] = freq.get(i, 0) + 1

Use:

Counter(data)
With Strings
c = Counter("python")
print(c)

Output:

{'p':1, 'y':1, 't':1, 'h':1, 'o':1, 'n':1}
Important Methods
1. most_common()
c = Counter([1,1,2,2,2,3])
print(c.most_common(2))

Output:

[(2, 3), (1, 2)]
2. elements()
c = Counter([1,2,2,3])
print(list(c.elements()))

Output:

[1, 2, 2, 3]
3. Update Counter
c = Counter([1,2])
c.update([2,3])
print(c)

Output:

{2:2, 1:1, 3:1}
4. Subtract
c1 = Counter([1,2,2])
c2 = Counter([2])

c1.subtract(c2)
print(c1)

Output:

{2:1, 1:1}
Access Values
c = Counter([1,2,2,3])

print(c[2])   # 2
print(c[5])   # 0 (no error)
Why Use Counter?
1. Frequency Counting

Count duplicates easily

2. Cleaner Code

No manual dictionary logic

3. Fast & Efficient

Optimized implementation

Real-Time Use Cases
Word count in text processing
Log analysis
Finding most frequent items
Data preprocessing (ML)
Interview One-Line Answer

Counter is a dictionary subclass used to count frequency of elements in a collection.

Quick Comparison

Counting	Manual	Automatic
Default value	Error	0
Built-in methods	No	Yes

What is Ordered Dictionary in Python?

An Ordered Dictionary (OrderedDict) is a special dictionary from Python’s collections module that remembers the order in which keys are inserted.

Simple Meaning

Normal dictionary: keeps data as key-value pairs
OrderedDict: keeps key-value pairs + insertion order guaranteed

Import OrderedDict
from collections import OrderedDict
Basic Example
from collections import OrderedDict

od = OrderedDict()

od["a"] = 1
od["b"] = 2
od["c"] = 3

print(od)

Output:

OrderedDict([('a', 1), ('b', 2), ('c', 3)])
Key Feature
Maintains insertion order
od = OrderedDict()
od["x"] = 10
od["y"] = 20
od["z"] = 30

Always stays:

x → y → z
What happens in normal dict?

(Python 3.7+ also preserves order, but OrderedDict has extra features)

d = {"a":1, "b":2, "c":3}
Important Methods in OrderedDict
1. move_to_end()

Move a key to end or beginning

od = OrderedDict(a=1, b=2, c=3)

od.move_to_end("a")
print(od)

Output:

b → c → a
2. popitem()
od.popitem(last=True)  # removes last item
Comparison
Feature	dict	OrderedDict
Order kept	Yes (3.7+)	Yes
Extra methods	No	Yes
Reordering	No	Yes
Use today	Mostly yes	Rare (legacy + special cases)
Why OrderedDict was important?

Before Python 3.7:

Normal dict did NOT maintain order
OrderedDict was required

Now:

dict keeps order automatically
OrderedDict used for advanced operations
Real-Time Use Cases

✔ LRU Cache implementation
✔ Maintaining sequence-sensitive data
✔ Processing logs in order
✔ Data pipeline ordering logic

Interview One-Line Answer

OrderedDict is a dictionary subclass that preserves the order of key insertion and provides extra methods for reordering elements.

Maintains insertion order
Has extra methods like move_to_end()
